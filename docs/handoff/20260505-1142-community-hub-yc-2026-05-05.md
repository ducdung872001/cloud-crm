### 3. Members module ⭐ MỚI [P0] — toàn bộ schema yêu cầu khắt khe

#### Bảng `members`

```
id              UUID PK
personal_seq    VARCHAR(8)  UNIQUE   -- STT cá nhân, cấp tự động bằng SEQUENCE/atomic counter
group_seq       INT                  -- 20 người/nhóm, derived: ceil(personal_seq / 20)
rank            VARCHAR(2)           -- 1 chữ số đầu của personal_seq (cached)
member_code     VARCHAR(40) UNIQUE   -- canonical = "${personal_seq}-${group_seq}", indexed
master_code     VARCHAR(40) UNIQUE NULL  -- "master-N" nếu là trưởng nhóm
full_name       VARCHAR(255)
phone           VARCHAR(20)  indexed
email           VARCHAR(255) NULL
occupation      VARCHAR(255) NULL    -- "công việc hiện tại" (Ngọc note)
avatar_url      TEXT NULL
birthday        DATE NULL
gender          VARCHAR(10) NULL     -- male/female/other
address         TEXT NULL
password_hash   VARCHAR(255) NULL    -- bcrypt
last_login_at   TIMESTAMPTZ NULL
status          VARCHAR(20)          -- active/suspended/graduated/inactive
joined_at       TIMESTAMPTZ
source          VARCHAR(20)          -- event_signup/manual/import/api
bsn_id          INT NOT NULL         -- TENANT SCOPE — per-tenant, KHÔNG share cross-tenant
created_at, updated_at  TIMESTAMPTZ
```

**Quy tắc nghiệp vụ (yc khách 5/5)**:
- `personal_seq` cấp tự động (SEQUENCE/counter atomic), KHÔNG để FE chỉ định.
- `group_seq = ceil(personal_seq / 20)` (default rule). 20 người/nhóm.
- KHÔNG phân nhóm theo địa lý — chỉ theo thứ tự thời gian gia nhập.
- Khi tạo member thứ 21 của 1 nhóm → tự khởi tạo nhóm mới.
- Hiện đang lên đầu hạng `6` (thành viên chính thức) — các đầu khác chưa rõ (xem mục "câu hỏi mở").

#### Bảng `member_history` (timeline) [P0]

Mọi sự kiện trong đời thành viên link về mã định danh.

```
id                   UUID PK
member_id            UUID FK → members  -- indexed: (member_id, occurred_at DESC)
member_code          VARCHAR(40)        -- denorm cho query nhanh; indexed
kind                 VARCHAR(30)        -- enum (xem dưới)
title                VARCHAR(255)
description          TEXT NULL
amount_vnd           BIGINT NULL
rating               SMALLINT NULL      -- 1-5
ref_event_id         UUID NULL
ref_registration_id  UUID NULL
ref_service_id       UUID NULL
ref_product_id       UUID NULL
variant              VARCHAR(120) NULL  -- size/màu cho product_bought
ref_status           VARCHAR(60) NULL   -- "đã đăng ký"/"đã mua"/"đã hủy" (Ngọc note)
occurred_at          TIMESTAMPTZ        -- yc khách: click tháng/ngày → filter timeline
bsn_id               INT NOT NULL
created_at           TIMESTAMPTZ
created_by           VARCHAR(100) NULL
```

`kind` enum: `event_checkin` / `service_used` / `product_bought` / `course_completed` / `role_issued` / `payment_in` / `debt_recorded` / `debt_settled` / `rating_given` / `note`.

**Auto-append history** khi:
- `event_registration` chuyển sang `confirmed` (kèm `member_code`) → kind=`event_checkin` (nếu đã check-in) hoặc `note`.
- `event_registration.payment_proof.status='approved'` → kind=`payment_in`, `amount_vnd=total_amount`.
- Admin issue role code mới → kind=`role_issued`.
- Service usage tracked (FE `CheckinServiceTracker` đã có) → kind=`service_used`.

### 6. Dynamic Fields — type `multi_select` [P1]

FE đã thêm type `multi_select` (size áo / màu áo / multi-choice — yc khách).

- BE không cần đổi schema (`dynamic_fields` đã store JSON).
- User submit: `dynamicFieldValues[fieldId]` cho `multi_select` lưu chuỗi `"opt1||opt2||opt3"` (delimiter `||`).
- BE giữ nguyên dạng đó. Khi export Excel → tách `||` để liệt kê.

## Ràng buộc & gợi ý kiến trúc

- **Microservice owner**: `cloud-market-master` (KHÔNG tách mới).
- **Tenant scope**: tất cả query MUST filter `bsn_id` — `members`, `member_history`, `event_comments` per-tenant, KHÔNG share cross-tenant.
- **Auth**: Bearer JWT, header `Authorization`. Header `Hostname` (đã có pattern).
- **Response wrapper**: `{ code: 0, result: {...}, message?: string }` (như existing).
- **Datetime**: ISO 8601 với timezone (`+07:00` hoặc `Z`).
- **Member code lookup**: index trên `members.member_code` + `master_code` để by-code lookup nhanh.
- **History performance**: composite index `(member_id, occurred_at DESC)`.
